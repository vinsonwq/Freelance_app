## 一、产品概述

### 1.1 产品名称
**自由派**（备选：项目排期助手、自由记账本）

### 1.2 产品定位
面向自由职业者、独立设计师、兼职人员的轻量级工具，将**项目日程排期**与**财务收款管理**合二为一，帮助用户一站式管理"什么时候干活、收了多少钱"。

### 1.3 目标用户
| 用户类型 | 典型场景 |
|---------|---------|
| 自由职业者 | 多个项目并行，需要跟踪每个项目的日程和收款 |
| 独立设计师 | 按项目收款，需要记录每次交付日期和对应款项 |
| 兼职人员 | 兼职日程与主职冲突管理，记录兼职收入 |
| 小微企业主 | 简单项目管理，不需要复杂的 ERP 系统 |

### 1.4 核心价值
- **一体化**：日程和财务在同一 App 内，不需要切换多个工具
- **轻量级**：专注于核心场景，不做复杂的功能堆砌
- **本地化**：所有数据存储在本地，不需要联网，保护隐私

## 三、信息架构

### 3.1 底部导航（4 个 Tab）
```
┌──────────┬──────────┬──────────┬──────────┐
│  📅 日程  │  📋 列表  │  📊 统计  │  👤 我的  │
│           │           │           │  (设置)   │
└──────────┴──────────┴──────────┴──────────┘
```

### 3.2 页面清单
| 页面 | 功能入口 | 说明 |
|-----|---------|------|
| 日程页 | Tab 1 | 日历月视图 + 日期格子展示项目 |
| 列表页 | Tab 2 | 项目台账，支持多维筛选 |

```

      → 如果项目名称不存在：创建新项目记录
```

## 四、功能需求

### 4.1 日程页（Calendar）

#### 4.1.2 添加日程
- **触发方式**：点击日期格子
- **交互形式**：底部弹窗（Modal Bottom Sheet）
- **表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| 项目名称 | 文本输入 + 自动联想 | ✅ | 输入时自动联想已有项目，选择后自动填充其他字段 |
| 服务日期 | 日期选择（多选） | ✅ | 默认包含点击的那个日期，可追加其他日期 |
| 客户名称 | 文本输入 | ❌ | |
| 项目类型 | 下拉选择 | ❌ | 可配置选项（默认：设计/开发/咨询/其他） |
| 项目金额 | 数字输入 | ❌ | 单位：元（RMB），V1 首次创建后不可修改 |
| 已收款 | 数字输入 | ❌ | 单位：元（RMB），V1 首次创建后不可修改 |
| 是否结清 | 开关 | ❌ | V1 首次创建后不可修改 |
| 备注 | 文本输入（多行） | ❌ | |

- **保存逻辑**：
  - 如果 `项目名称` 已存在（数据库 `UNIQUE` 约束）：**追加**服务日期到已有记录的 `schedule_dates` 字段
  - 如果 `项目名称` 不存在：创建新记录
  - **V1 约束**：`项目金额`、`已收款`、`是否结清` 首次创建后不可修改（V1.1 才支持编辑）

## 五、数据模型

### 5.1 核心设计决策

> ⚠️ **重要**：以下设计决策已在 V1.0.1 中确定，V1 不再变更。

1. **一个项目 = 一条数据库记录**（不是每日一条记录）
2. **`schedule_dates` 字段用 JSON 数组存储多个日期**，如 `["2026-06-03","2026-06-05","2026-06-10"]`
3. **金额字段首次创建后不可修改**（V1 约束）：
   - `total_amount`（项目金额）
   - `received_amount`（已收款）
   - `is_settled`（是否结清）
   - *V1.1 才支持编辑这些字段*
4. **`project_name` 设为 `UNIQUE`**：相同项目名称自动追加日期，不创建重复记录
5. **V1 不做云同步**：所有数据存储在本地 SQLite

### 5.2 数据库表结构

```sql
CREATE TABLE tb_project (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  project_name TEXT NOT NULL UNIQUE,          -- 项目名称（唯一）
  schedule_dates TEXT NOT NULL,               -- 服务日期（JSON 数组）
  client_name TEXT,                           -- 客户名称
  project_type TEXT,                          -- 项目类型
  total_amount REAL DEFAULT 0.0,             -- 项目金额
  received_amount REAL DEFAULT 0.0,          -- 已收款金额
  is_settled INTEGER DEFAULT 0,              -- 是否结清（0=未结清，1=已结清）
  remarks TEXT,                               -- 备注
  created_at INTEGER NOT NULL                 -- 创建时间（毫秒时间戳）
)
```

### 5.3 Dart 数据模型（`project.dart`）

```dart
class Project {
  int? id;
  String projectName;
  List<String> scheduleDates;  // JSON 数组解析为 List<String>
  String? clientName;
  String? projectType;
  double totalAmount;
  double receivedAmount;
  bool isSettled;
  String? remarks;
  DateTime createdAt;

  // 计算属性
  bool get isOverdue;          // 所有日期都已过，且未结清
  double get pendingAmount;    // totalAmount - receivedAmount
}
```

- ✅ 提供了 `install_flutter.ps1` 自动安装脚本（Windows PowerShell）

### 9.2 手动安装步骤
1. 下载 Flutter SDK：  
   `https://storage.flutter-io.cn/flutter_infra_release/releases/stable/windows/flutter_windows_3.32.8-stable.zip`
2. 解压到 `C:\flutter`（不要有空格或中文路径）
3. 添加 `C:\flutter\bin` 到系统 `PATH`
4. 设置国内镜像环境变量：
   ```
   FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn
   PUB_HOSTED_URL=https://pub.flutter-io.cn
   ```
5. 运行 `flutter doctor` 验证安装
6. `cd freelance_app` → `flutter pub get` → `flutter run`

### 9.3 依赖包清单（`pubspec.yaml`）
